This command sets the author name and email address to be used with your commits. Git config is also used in other scenarios.
Or you can directly change in file by this command

create repository, means the project which can be access from the git. So to make folder git repo write below command.
git init
this will create a empty and hidden repository (.git) in the project folder. when we run this command a default branch "master" is created.


git status
this will show you the status of file, kuch change hua ho etc.


git add your-file or folder name
this will use to add particular files/folder

git add .
this will use to add all files/folder


Now ab agr mene kisi file mai change kiya hai toh first mujhe us file ko add krna pdega using git add command than fir usko commit krenge


git commit -m " Commit Message"
This command changes the head. It records or snapshots the file permanently in the version history with a message. Jo change kiya ho vo message mai likh do (-m means message)


git log
This command is used to check the commit history. By default, if no argument passed, Git log shows the most recent commits first. We can limit the number of log entries displayed by passing a number as an option, such as -3 to show only the last three entries.


suppose we want to keep some files private so that git can't able to access those files. so for this create one file (.gitignore) file and under this file write the name of the file which you want to hide from git for example ignorefile.html. you can also hide .gitignore file by writing .gitignore in .gitignore file


git branch
it is used to see all the branch, it is used for production and development for example multiple people are working on one project, so that we can create the branch for it.. so one developer work on that branch and other one on other branch. so the code can not get conflict. before moving to new branch we must move to master branch.. so that we get the main code.

git branch branch_name
it is used to create a new branch

git branch --delete branch_name
used to delete the branch, but if we delete the branch the changes we've done under that branch will all go.


git checkout
this is used to move from one commit to other commit and also to move to the other branch.



git merge branch_name
